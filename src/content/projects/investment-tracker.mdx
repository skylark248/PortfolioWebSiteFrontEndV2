At NAV Fund Administration Group I led the backend of this system — a Spring Boot + Kafka + Cassandra + React stack that gave fund administrators, ops teams, and management a single live view of investment document state across **5,000+ funds**. Before it existed, the same team reconciled three disjoint systems by hand every day.

Before the platform existed, document workflows were distributed across a Document Management System, external fund data providers, and an internal accounting system that operated on separate schedules. Analysts reconciled state manually every day. When quarter-end processing fell behind, nobody knew until it was already late.

The platform replaced that fragmented picture with a real-time view. Spring Boot schedulers polled the Document Management System on configurable intervals, normalized document state events, and published them to Kafka topics. Downstream consumers processed those events, applied business rules around state transitions and stalled-document detection, and wrote to both a SQL Server transactional store and a Cassandra reporting tier optimized for the dashboard query patterns. A React frontend — built by a separate team against API contracts I defined and maintained — presented fund-level and portfolio-level dashboards to 2,500+ daily users.

As lead engineer on the backend, I owned the service architecture, the Kafka pipeline design, the database schema decisions (including the choice to introduce Cassandra for the read layer), and the API contracts. I also coordinated with operations stakeholders to translate reporting requirements into data model decisions that could be answered efficiently at query time.

The outcome: document processing time fell 40%, the manual daily reconciliation was eliminated, and the platform scaled to 10M+ document state records without impacting transactional write performance.
